# 低照度候选集人工标注工作台设计

日期：2026-07-16

## 1. 背景与目标

当前候选集 `datasets/low_light_camera_full_rgb_v2_candidates` 含 388 张来自
`camera_full_rgb.mp4` 前五分钟的原始候选帧，其中包括 300 张逐秒基础帧和
88 张运动、场景切换或光照变化事件帧。旧版 60 组低照度图片与标签已从正式训练集
隔离，不参与本轮标注或后续训练。

本设计建立一个只在当前电脑使用的人工标注工作台，使全部 388 张候选均经过明确的
人工判断，并输出可验证、可审计、尚未合并的 `pic+label` 数据包。系统不得自动批准
样本、不得自动合并正式训练集、不得启动模型训练。

## 2. 已确认的标注原则

1. 标注对象是机器人实际需要收卷并跟随的目标水带，不按颜色限定。其他软管、线缆、
   非目标水带和相似长条物体均为干扰物。
2. 全部 388 张候选必须逐张人工确认。系统建议仅作为草稿，不能代替人工批准。
3. 短距离遮挡且遮挡前后走向明确时，可平滑插值中心线；长遮挡、目标身份不确定或
   走向无法可靠判断时，不得猜测，必须进入二次复核或排除。
4. 原始未增强候选帧是正式训练图片。增强结果只在标注页面中辅助观察，不作为独立
   训练样本，也不得替换原图。
5. 主方案是本机专用标注页面；页面确实不可用时，使用 LabelMe 作为备选，但导回后
   仍执行完全相同的审核与导出门禁。

## 3. 范围

### 3.1 本轮包含

- 候选集完整性检查。
- 模型、颜色线索和相邻帧辅助的预标注草稿。
- 本地人工标注、分类、自动保存和修改历史。
- 困难负样本分类与全零标签生成。
- LabelMe 导出、导入和一致性校验备选路径。
- 二次复核队列、完整验证、overlay 总览和可合并数据包导出。
- 页面、存储、坐标转换、门禁和 LabelMe 往返测试。

### 3.2 本轮不包含

- 将导出包合并到 `datasets/newdata`。
- 修改训练/测试划分。
- 启动训练、微调或导出 ONNX。
- 使用增强图片替换原始训练图片。
- 恢复已隔离的旧 60 组数据。

## 4. 总体架构

系统由五个边界清晰的组件组成。

### 4.1 启动与健康检查

启动器默认绑定 `127.0.0.1:8765`，禁止绑定 `0.0.0.0`。若端口被占用，按顺序尝试
`8766` 至 `8799`，并在终端显示实际 URL。启动器在提示用户访问前调用 `/health`，
确认服务、候选目录、清单和标注状态均可用。失败时输出诊断日志，不声称页面可访问。

### 4.2 预标注器

预标注器读取原图，结合当前可用模型、颜色锚点、候选时间顺序和相邻帧信息生成建议
中心线、置信度与冲突标记。建议不能改变人工状态。取得此前稳定运行的旧模型后，可把
它作为第二路意见加入困难样本审计，但旧模型缺失不阻塞工作台实现或人工标注。

### 4.3 本地标注页面

页面展示原始 640×480 图像、增强预览、前后相邻候选帧、建议锚点和最终中心线。
用户可添加、拖动、删除锚点，并使用撤销、重做、清空和预览切换。页面实时显示最终
二值标签的覆盖效果。草稿自动保存，但最终样本类别必须由用户主动确认。

### 4.4 审核存储

候选图片和原始 `manifest.csv` 始终只读。标注状态使用临时文件加原子替换保存；修改
历史采用追加记录，包含时间、前后状态、前后锚点和来源。状态文件记录原图 SHA-256，
防止候选图片被替换后继续沿用旧标签。

### 4.5 验证与导出

验证器只接受已经完成人工确认和必需二次复核的记录。通过后导出到
`datasets/low_light_camera_full_rgb_v2_reviewed`，但不触碰正式训练集。

## 5. 页面工作流

### 5.1 队列与导航

顶部显示总数以及未处理、正样本、困难负样本、排除和待二次复核数量。用户可按状态、
事件类型、低置信度、模型冲突和干扰物类别筛选。页面支持上一张、下一张和按时间跳转，
离开当前图像前先完成自动保存。

### 5.2 样本状态

每张候选最终只能属于以下一种状态：

- `positive`：可确认目标水带，至少有三个有效锚点，并生成中心线标签。
- `hard_negative`：没有目标水带，但存在易误识别结构，生成全零标签并进入训练包。
- `excluded`：严重模糊、严重曝光异常、近重复、目标身份不明或长遮挡无法判断，
  不生成训练对。
- `needs_second_review`：暂时不能确定的中间状态，不能通过最终门禁。

### 5.3 困难负样本类别

困难负样本至少选择一个干扰物标签：

- `floor_seam_or_edge`
- `shadow_or_reflection`
- `cable_or_rope`
- `non_target_hose`
- `elongated_object`
- `motion_or_low_light_artifact`

困难负样本必须人工确认画面中不存在机器人作业目标水带；若目标水带存在但难以标注，
不能将其误归为负样本。

### 5.4 排除原因

排除样本必须选择原因：`near_duplicate`、`severe_motion_blur`、
`severe_exposure_failure`、`ambiguous_target_identity`、
`long_occlusion_unknown_path` 或 `no_training_value`。排除不会删除候选源文件。

### 5.5 防误操作

撤销、重做、切换预览和导航可配置快捷键；正样本、困难负样本和排除不提供单键批准。
最终确认需要显式按钮，并显示即将保存的类别。页面刷新或意外关闭后从最后一次原子保存
恢复。

## 6. 中心线与标签约束

人工锚点使用原始图像坐标，`x` 在 `[0, 639]`，`y` 在 `[0, 479]`。同一 `y` 只保留
一个锚点，按 `y` 严格递增。正样本不少于三个锚点，通过单调形状保持插值生成中心线。
输出标签为 640×480、单通道 PNG，像素值仅为 `0` 或 `1`，每行最多一个正像素。

短遮挡插值必须由用户确认；出现长遮挡、路径分叉或目标身份变化时禁止跨区推测。困难
负样本输出相同格式的全零 PNG。排除样本不输出图片或标签到最终训练对目录。

## 7. 二次复核策略

以下样本必须完成第二次人工复核：

- 所有困难负样本。
- 人工对系统建议作出明显修改的正样本。
- 存在遮挡、低置信度或多路建议冲突的样本。
- 目标附近包含线缆、非目标水带、地面接缝或相似长条物体的样本。
- 所有曾进入 `needs_second_review` 的样本。

其余普通正样本按固定随机种子抽查 10%。第二次复核保存独立时间与结果；未通过的样本
退回编辑队列。

## 8. LabelMe 备选

当本地页面在端口回退和健康检查后仍不可用时，导出 LabelMe 工作目录。正样本使用
`target_hose_centerline` 折线；困难负样本使用图像级 flags 记录类别且不画目标折线；
排除状态与原因也写入 flags。导入器拒绝未知标签、多个目标中心线、越界点、少于三个点
的正样本以及缺失状态的 JSON。

LabelMe 导入只更新审核草稿，不能直接生成合并结果。导入后仍需执行项目内部二次复核、
标签栅格化和最终门禁。

## 9. 数据文件与产物

工作目录位于候选集之外，避免与源图片混合。核心文件包括：

- `annotation_state.json`：当前人工状态与锚点。
- `annotation_history.jsonl`：不可覆盖的修改历史。
- `preannotation.json`：系统建议及来源。
- `review_report.json`：首次确认、二次复核和抽查统计。
- `validation_report.json`：最终门禁结果。

通过后输出目录 `datasets/low_light_camera_full_rgb_v2_reviewed` 包含：

- `pic/`：原始、未增强、已获准进入训练的数据图片。
- `label/`：正样本中心线标签和困难负样本全零标签。
- `overlay/`：逐图检查叠加结果。
- `annotation.json`：最终结构化标注记录。
- `review_report.json` 和 `validation_report.json`。
- 分页 overlay 接触表。

## 10. 最终门禁

导出必须同时满足：

1. 388 张候选全部具有明确最终状态，不存在 `needs_second_review` 或未处理记录。
2. 所有必需二次复核与 10% 普通样本抽查完成。
3. 所有源图哈希与候选 `manifest.csv` 一致。
4. 正样本和困难负样本的图片、标签、记录 stem 完全匹配。
5. 所有标签尺寸、模式、像素值和单行单点约束通过。
6. 正样本标签非空；困难负样本标签全零；排除样本不出现在训练对中。
7. 导出图片与原始候选哈希一致，增强预览未混入。
8. 无文件名冲突、重复内容冲突或正式测试集哈希泄漏。
9. overlay 与接触表生成成功，验证报告无错误。

任何一项失败都阻止导出为“可合并”状态。即使全部通过，也必须等待用户单独批准后才能
合并正式训练集。

## 11. 错误处理

- 端口冲突：自动顺延端口并报告实际 URL。
- 服务启动失败：保留诊断日志并提供 LabelMe 导出命令。
- 保存失败：不改变内存中的已确认状态，显示错误并允许重试。
- 状态文件损坏：拒绝覆盖，保留损坏副本并从最近有效原子版本恢复。
- 候选哈希变化：停止对应样本标注并要求重新建立草稿。
- LabelMe 格式错误：逐文件报告，不部分导入错误记录。
- 验证失败：保持工作目录可编辑，不发布不完整导出包。

## 12. 测试策略

实现需包含：

- 锚点坐标、插值和二值栅格化单元测试。
- 正样本、困难负样本、排除和待复核状态转换测试。
- 原子保存、历史追加与损坏恢复测试。
- 候选哈希与增强图误混入阻断测试。
- 端口占用、端口回退和 `/health` 测试。
- LabelMe 导出/导入往返和非法 JSON 拒绝测试。
- 388 张覆盖、二次复核、10% 抽查和最终门禁测试。
- 使用合成候选的小规模页面 API 端到端测试。
- 导出包与正式训练/测试集重名、重复哈希和泄漏测试。

## 13. 完成标准

设计实现完成的标准是：本地页面能够在当前环境启动并通过健康检查；若不可用，LabelMe
往返路径可工作；草稿和审核状态可恢复；自动测试通过；但候选标注、最终导出、正式合并
和模型训练仍分别受人工操作与后续用户指令控制。
