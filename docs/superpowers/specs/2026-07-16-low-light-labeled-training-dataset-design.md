# 低照度水带完整训练数据生成与合并设计

日期：2026-07-16  
状态：已实施，待训练

## 1. 目标

从 `E:\New_DOTEK\数据\camera_full_rgb.mp4` 的前五分钟 `[0, 300)` 中，从 0 秒开始每 5 秒抽取一帧，共 60 帧（0、5、…、295 秒）。对每帧进行低照度增强和水带中心线标注，形成 60 组可直接被现有训练器加载的 `JPG + PNG` 配对数据，并安全合并进 `datasets/newdata/train`。

本任务完成后，训练集应由 4858 组增加到 4918 组；测试集保持 867 组不变。本任务不启动训练。

## 2. 输入与现有约束

- 视频：640×480、约 22.128 FPS、时长约 384.63 秒；
- 训练图像：640×480 JPG；
- 标签：640×480 单通道 PNG，像素值只能为 0 或 1；
- 标签语义：约 1 像素宽的水带中心线，而非水带区域分割；
- 当前训练加载器分别排序 `pic/*.jpg` 与 `label/*.png`，因此必须保证图片与标签文件名主干一一对应；
- 当前模型输入为 384×288、18 个行锚点、50 个网格、1 条水带；
- 初始标注模型使用现有 `logs/0209_1509_lr_1e-04_b_64/best_model.pth`。

## 3. 方案选择

### 3.1 未采用：纯模型伪标签

速度最快，但低照度域偏移可能造成断裂、偏移或错误中心线，未经检查直接进入训练集会放大模型偏差。

### 3.2 未采用：完全手工逐像素标注

可控性高，但效率低，且逐像素绘制难以保持与现有单像素中心线标签一致。

### 3.3 采用：模型辅助标注、规则化修正和逐帧审核

现有模型提供 18 个初始锚点；程序融合原始图和增强图预测，并以红色水带连通域中心作为结构化辅助候选，依据置信度进行异常点剔除、连续性约束和逐行插值，再通过叠加图逐帧检查并修正。该流程兼顾效率、标签一致性与低照度场景质量。

## 4. 处理架构与数据流

```text
视频
  -> 60个固定时间戳抽帧
  -> 原始帧质量统计
  -> 自适应Retinex-LAB增强
  -> 原图/增强图双路模型推理
  -> 红色水带连通域辅助候选
  -> 多源置信度融合与锚点规则化
  -> 640×480单像素中心线PNG
  -> 逐帧叠加审核与必要修正
  -> 配对/格式/泄漏/哈希质量门禁
  -> 原子合并到newdata/train
  -> 合并清单与校验报告
```

处理过程目录为：

```text
datasets/low_light_camera_full_rgb/
├── original/          # 60张原始抽帧
├── enhanced/          # 60张最终候选训练图
├── label_initial/     # 模型辅助生成的初始标签
├── label/             # 审核后的60张最终标签
├── overlay/           # 60张标签叠加审核图
├── manifest.csv       # 抽帧、增强、预测、审核指标
├── annotation.json    # 最终锚点与修正记录
├── preview.jpg        # 原图/增强图对照
└── merge_receipt.json # 正式合并记录和SHA-256
```

## 5. 抽帧与图像格式

- 时间戳固定为 `range(0, 300, 5)`，严格得到 60 个值；
- 文件主干为 `lowlight_camera_full_rgb_tXXXXXX`，例如 `lowlight_camera_full_rgb_t000000` 与 `lowlight_camera_full_rgb_t000295`；
- 原始帧和增强帧均保持 640×480、8-bit、三通道；
- JPG 保存质量为 95；训练加载器继续负责缩放到 384×288 和灰度化；
- 每帧记录目标时间戳和实际读取位置；任一目标帧读取失败时不发布不完整结果。

## 6. 自适应低照度增强与复原

1. 测量均值、中位数、5%/95%分位、暗像素比例、高光比例和Laplacian方差；
2. 使用带增益限制的灰度世界白平衡；
3. 转换到LAB，在L通道执行自适应保边降噪；
4. 使用大尺度照明估计和受限Retinex恢复暗部反射细节；
5. 根据中位亮度计算受限Gamma，将暗场目标亮度推向约115；
6. 使用温和CLAHE改善局部对比度；
7. 对输入高亮区域和输出分位数实施高光保护；
8. 根据暗像素比例和清晰度执行条件反锐化；
9. 较亮帧只进行弱增强，避免240–295秒画面过曝。

增强后必须保持尺寸、通道和像素类型不变。高光裁切明显恶化时自动降低增强混合权重。

## 7. 模型辅助标注

### 7.1 双路预测

- 对原始图和增强图使用相同模型与预处理分别推理；
- 对每个行锚点保留前景位置、前景概率、背景概率及原图/增强图位置差；
- 两路位置接近时按置信度加权融合；位置冲突时选择高置信度结果并标记为需要重点审核；
- 背景概率过高的锚点不强制生成水带点。
- 对可辨识的红色水带提取连通域逐行中心点，作为低照度域偏移时的辅助候选；该结果仍须逐帧审核，不直接替代人工判断。

### 7.2 锚点规则化

- 至少需要 3 个有效锚点才能形成初始中心线；
- 使用相邻锚点位移和局部中位趋势剔除明显离群点；
- 对短缺口进行形状保持插值，不跨越长距离无依据外推；
- 保持水带轨迹连续，同时允许真实弯曲；
- 记录原始锚点、融合锚点、被剔除点和最终锚点，保证可追溯。

### 7.3 密集中心线生成

- 将最终锚点映射回640×480坐标；
- 在有效纵向范围内逐行插值x坐标；
- 每行最多写入一个值为1的像素，其余为0；
- 不对不可见区域进行无依据延伸；
- 经逐帧审核确认目标水带不可见时，允许显式标记为负样本并生成全0标签；
- 标签使用无损PNG保存。

## 8. 逐帧审核与修正

- 为每帧生成增强图上的彩色中心线叠加图；
- 自动检查有效锚点数、纵向覆盖、最大局部跳变、原图/增强图预测差异和平均置信度；
- 任何低置信度、断裂、跳变或明显偏离水带的结果必须修正最终锚点后重新生成标签；
- 所有60帧均需检查，不只检查自动告警帧；
- `annotation.json` 记录每帧审核状态和是否发生修正；
- 只有审核通过的60组数据才能进入合并步骤。若仍有未通过项，合并整体停止。

## 9. 正式训练集合并

合并前执行以下门禁：

1. `enhanced` 与 `label` 均恰好60个文件；
2. JPG与PNG主干集合完全一致；
3. 所有图像和标签尺寸均为640×480；
4. 所有标签仅包含0和1；目标可见帧包含有效中心线，显式审核的不可见负样本使用全0标签；
5. 新文件名不与训练集或测试集冲突；
6. 新图像SHA-256不与测试集图像相同；
7. 现有训练集4858组的文件名与SHA-256快照未变化；
8. 当前训练集原有图片和标签主干集合一致。

合并过程先把60组文件写入训练目录内的临时文件名，全部写入和校验成功后再原子重命名为最终文件名。若中途失败，只清理本次创建的临时文件，不改动原有文件。

合并后必须满足：

- `datasets/newdata/train/pic`：4918张JPG；
- `datasets/newdata/train/label`：4918张PNG；
- 两侧主干集合完全相同；
- `datasets/newdata/test` 仍为867张图像和867张标签；
- 60个新主干不出现在测试集；
- `merge_receipt.json` 保存合并前后数量、每个新文件的SHA-256及目标路径。

## 10. 错误处理与幂等性

- 视频不存在、时长不足、模型不兼容、任一帧失败或任一质量门禁失败时返回非零退出码；
- 默认拒绝覆盖已经生成的过程数据；
- 已合并且哈希一致时，重复执行报告“已存在且一致”，不重复追加；
- 同名但哈希不同视为冲突，停止并报告；
- 不删除或覆盖人工标签；
- 不修改测试集；
- 不运行任何训练、微调或模型导出命令。

## 11. 测试与验收

实现采用测试驱动方式，覆盖：

1. 时间戳为0–295秒、步长5秒、总数60；
2. 增强参数范围、输出格式、暗图提亮和亮图高光保护；
3. 模型输出维度及18锚点映射；
4. 双路预测融合、离群点删除、短缺口插值和逐行单像素标签生成；
5. 标签像素值、尺寸、逐行单点、可见帧非空、审核负样本为空和主干配对；
6. 低质量标注阻止合并；
7. 名称冲突、哈希冲突和测试集泄漏阻止合并；
8. 临时文件发布失败不影响原有4858组数据；
9. 合并后训练集4918/4918、测试集867/867；
10. 环境中没有训练进程，项目日志和模型权重不发生训练写入。

## 12. 非目标
