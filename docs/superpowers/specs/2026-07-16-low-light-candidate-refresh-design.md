# 低照度旧样本隔离与候选数据重抽取设计

## 目标

在不删除任何历史材料、不生成新标签、不启动训练的前提下：

1. 将已合并进 `datasets/newdata/train` 的旧版60组低照度图片和标签事务性隔离，使正式数据集恢复为4858组训练数据和867组测试数据。
2. 从 `E:\New_DOTEK\数据\camera_full_rgb.mp4` 前五分钟生成更丰富、可追溯、适合人工审核的新候选图片集。

## 约束

- 旧60组只能移动到隔离区，禁止删除。
- 隔离前必须按 `merge_receipt.json` 校验每个源文件的 SHA-256；有任一不一致立即停止且不移动文件。
- 隔离操作必须先复制到临时目录、验证副本，再移除正式训练集源文件，最后原子重命名临时目录。
- 隔离完成后训练集必须是4858张图片和4858张标签，测试集必须保持867/867，图片和标签 stem 必须完全配对。
- 新候选数据禁止复用旧 `lowlight_camera_full_rgb_` 前缀，使用 `camera_full_rgb_v2_t#########`（毫秒时间戳）。
- 候选阶段只生成图片、指标、原因、预览和清单，不生成训练标签，不写入 `datasets/newdata`。
- 视频只处理 `[0, 300)` 秒。

## 旧60组隔离

隔离输出：

```text
datasets/quarantine/low_light_camera_full_rgb_v1/
├── train_pic/             # 从正式训练集隔离的60张增强图
├── train_label/           # 从正式训练集隔离的60张正式标签
├── source_bundle/         # 原低照度材料包的只读镜像清单（不复制大文件）
└── quarantine_receipt.json
```

`quarantine_receipt.json` 记录源/目标路径、SHA-256、执行前后数据集计数、原 `merge_receipt.json` 哈希以及完成时间。重复执行时，如果隔离已经完整完成则返回幂等成功；半完成状态必须拒绝继续并给出检查结果。

## 新候选抽取

### 解码与候选时间点

- 以0.25秒为分析步长，前五分钟共最多1200个分析时刻。
- 所有整数秒帧都是基础候选，共最多300张。
- 对每个非整数秒帧计算：
  - 与前一分析帧的灰度平均绝对差；
  - 稀疏光流中位位移；
  - 灰度均值及相邻亮度变化；
  - Laplacian 清晰度；
  - 暗像素比例；
  - 红色水带颜色锚点数量（只作为审核提示，不能作为标签）。
- 在每个1秒窗口内，除基础帧外最多增加一个事件分最高的帧。
- 事件分由归一化的光流、帧差和亮度突变组成；清晰度只用于清单和审核，不因模糊而自动剔除，因为真实运动模糊是重要困难样本。

### 去重

- 基础整数秒帧始终保留。
- 事件候选与已保留帧进行64位 dHash 比较；时间相距不超过1秒且 Hamming 距离不大于3时视为近重复并跳过。
- 目标候选量约300至600张；实际数量由视频事件决定，不通过复制或放宽去重凑数。

### 输出

```text
datasets/low_light_camera_full_rgb_v2_candidates/
├── frames/
├── contact_sheets/
├── manifest.csv
└── summary.json
```

`manifest.csv` 每行记录 stem、目标/实际时间、基础或事件来源、事件原因、帧差、光流、亮度、暗像素比例、清晰度、颜色锚点数、dHash。

`summary.json` 记录视频哈希、视频元数据、参数、候选总数、基础/事件数量、每项指标的分位数和输出文件哈希。

联系表按时间顺序每页20张，在图上显示时间、来源、亮度、光流和颜色锚点数，供下一阶段人工筛选正样本、动态样本和困难负样本。
