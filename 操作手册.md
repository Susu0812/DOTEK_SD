# Ultra-Fast Lane Detection 操作手册

## 一、数据准备

### 1.1 数据集目录结构

```
datasets/
└── newdata/
    ├── train/              # 训练集
    │   ├── label/          # 标签文件 (png)
    │   └── pic/            # 图像文件 (jpg/png)
    └── test/               # 测试集
        ├── label/          # 标签文件 (png)
        └── pic/            # 图像文件 (jpg/png)

```

### 1.2 列表文件格式

```
# train.txt / val.txt 格式
images/train/0001.jpg labels/train/0001.png
images/train/0002.jpg labels/train/0002.png
...
```

**注意**：路径使用相对路径，以数据集根目录为基准

## 二、训练模型

### 2.1 基本训练命令

```bash
python train_alian.py
```

### 2.2 关键参数说明

| 参数 | 说明 | 常用值 |
|------|------|--------|
| `--source` | 训练数据集路径 | `./datasets/train/` |
| `--val_source` | 验证数据集路径 | `./datasets/val/` |
| `--log_path` | 模型保存路径 | `./logs/` |
| `--epoch` | 训练轮数 | 100-200 |
| `--batch_size` | 批次大小 | 32-64 |
| `--learning_rate` | 学习率 | 1e-4 (Adam), 0.1 (SGD) |
| `--optimizer` | 优化器 | Adam / SGD |
| `--scheduler` | 学习率调度 | cos / multi |
| `--griding_num` | 网格列数 | 50 (小图) / 200 (大图) |
| `--row_anchor` | 行锚点数 | 18 |
| `--num_lanes` | 车道线数量 | 1-4 |
| `--use_aux` | 辅助分割任务 | True / False |

### 2.3 恢复训练

```bash
python train_alian.py \
    --source ./datasets/newdata/train/ \
    --val_source ./datasets/newdata/test/ \
    --resume ./logs/0209_1509_lr_1e-04_b_64/latest_model.pth
```

### 2.4 查看训练日志

```bash
# 实时监控
tensorboard --logdir=./logs

# 查看文本日志
cat ./train_log/train_epoch_summary.txt
```

## 三、模型测试

### 3.1 图片测试

```bash
python demo.py \
    --backbone 18 \
    --model ./culane_18.pth \
    --source ./datasets/pic \
    --savepath ./datasets/test2 \
    --griding_num 200 \
    --num_row_anchors 18 \
    --num_lanes 4
```

**参数说明**：
- `--model`: 训练好的模型路径 (.pth)
- `--source`: 测试图片文件夹路径
- `--savepath`: 检测结果保存路径
- `--griding_num`: 必须与训练时一致
- `--num_row_anchors`: 行锚点数
- `--num_lanes`: 车道线数量

### 3.2 修改行锚点

如果测试效果不佳，可能需要修改 `data/constant.py` 中的行锚点：

```python
# 根据你的数据集调整
my_row_anchor = [121, 131, 141, 150, 160, 170, 180, 
                 189, 199, 209, 219, 228, 238, 248, 
                 258, 267, 277, 287]
```

## 四、导出 ONNX

### 4.1 修改导出配置

编辑 `export_onnx.py`：

```python
# ========== 配置参数 ==========
cls_num_per_lane = 18      # 行锚点数，与训练时一致
griding_num = 50           # 网格数，与训练时一致
use_aux = False            # 必须设为 False

# 模型路径
test_model = './logs/0209_1509_lr_1e-04_b_64/latest_model.pth'

# 输出路径
output_path = 'lane_model.onnx'
# ==============================
```

### 4.2 执行导出

```bash
python export_onnx.py
```

导出成功后会显示：
```
模型已成功导出为ONNX格式: lane_model.onnx
ONNX模型验证通过！
```

### 4.3 导出参数说明

```python
torch.onnx.export(
    net,                    # 模型
    dummy_input,            # 示例输入 (1, 3, 288, 800)
    output_path,            # 输出路径
    export_params=True,     # 导出参数
    opset_version=13,       # ONNX算子版本
    do_constant_folding=True,  # 常量折叠优化
    input_names=['input'],
    output_names=['output'],
    dynamic_axes={          # 动态批次维度
        'input': {0: 'batch_size'},
        'output': {0: 'batch_size'}
    }
)
```

## 七、快速参考

### 7.1 常用命令速查

```bash
# 训练
python train_alian.py --source ./datasets/train/ --val_source ./datasets/val/ --epoch 200 --batch_size 64

# 恢复训练
python train_alian.py --source ./datasets/train/ --resume ./logs/xxx/latest_model.pth

# 测试
python demo.py --model ./logs/xxx/best_model.pth --source ./test_images/ --savepath ./results/

# 导出ONNX
python export_onnx.py

# ONNX推理
python inference_onnx6.py --model lane_model.onnx --image test.jpg

# INT8量化
python onnx_int8.py
```

### 7.2 参数对应关系

| 训练参数 | 导出参数 | 推理参数 |
|----------|----------|----------|
| `--griding_num` | `griding_num` | `griding_num` |
| `--row_anchor` | `cls_num_per_lane` | `num_row_anchors` |
| `--num_lanes` | - | `num_lanes` |
| `--use_aux` | `use_aux=False` | - |

**重要**：这些参数在训练、导出、推理三个阶段必须保持一致！

### 7.3 常见问题

| 问题 | 解决方案 |
|------|----------|
| 显存不足 | 减小 `--batch_size` 或 `--griding_num` |
| 检测不到车道线 | 检查行锚点配置是否匹配数据集 |
| ONNX导出失败 | 确保 `use_aux=False` |
| 推理结果不对 | 检查预处理是否与训练一致 |
